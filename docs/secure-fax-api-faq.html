Secure Fax API FAQ

If you can’t find an answer to your question please contact us.

  • Q: What is an API?
    A: An Application Programming Interface (API) is a set of routines, protocols, and tools for building software applications an API specifies how software components should interact. Additionally APIs are used when programming graphical user inteface (GUI) components. [wikipedia]
  • Q: What can we do with an API?
    A: An API can be “Consumed” by any number of websites, tools and systems. If you have an EMR or CRM you can integrate with our API to have a fax automatically send whenever you add a new customer or any other functionality. Your IT staff can integrate with our API and send/receive faxes from any application that supports custom integrations.
    • Q: In what languages is your API available?
  • A: Our API is written in a format called “REST”. This is an industry standard interface that most API’s comply with. We have samples in PHP, .NET, C#, Python and more. If you use Postman you can check out our custom Postman integration and generate code in dozens of other languages. Any language that can consume an API is compatible with our API.
  • Q: How do I send a fax with your API?
    A: Great question! We wrote an article that can get you started asap.
  • Q: We use an EMR. Can you integrate fax into our workflow?
    A: Yes, We can integrate into any EMR. It starts with something as simple as Print to Fax driver and we can also perform custom integration into your EMR using our HIPAA Compliant Secure Fax API. Please contact us for more information.
  • Q: How do I sign up for API access?
    A: Go here to learn more about our API and sign up here.
  • Q: Do I need to be a developer to use your API?
    A: Not necessarily. Some CRMs and EMR have plugins that allow for simple drag and drop API integrations. We can also assist with a custom integration on a case by case basis. Please contact us and let us know what you need and we’ll get right back to you.
  • Q: Can I get customer service help if I needt it?
    A: Yes. We provide a lot of documentation on the API ( here, here and here ) but should you need more documentation or help please reach out. We have live tech support based in the USA and real people answer the phones. Call us at 1-800-473-6208. We do charge for advanced integration support but we believe our documentation and samples should be enough to get you started.
  • Q: How secure is your HIPAA compliant API Service?
    A: We utilize the highest level of TLS encryption available for data in transit either through our secure website or secure API interfaces. We also use AES 256 bit encryption of ePHI data to guarantee privacy and prevent disclosure from intrusion. We support TLS protected SMTP email with optional REQUIRE TLS extension in accordance with the IETF RFC 3207. FTPS and SFTP with TLS for safe and secure transport of documents to and from your existing servers. Comprehensive HTTPS API allows for easy and flexible integration of systems.
  • Q: What are your acceptable document formats for API submission?
    A: Word, PDF, Tiff, Excel, Publisher and PowerPoint.
